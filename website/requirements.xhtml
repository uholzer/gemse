Here I collect requirements for the gemse. These are whishes from future users of gemse.

Basic features

  • Capturing and editing of the semantics of mathematical formulas
  • Support for OpenMath and (content) MathML

Various things

  • Navigation through content dictionary entries and signs must be fast.
  • Every feature should be accessible by mouse and by keyboard. (In hope that gemse will be fast with a pen on a tablet PC also.)
  • gemse must produce OpenMath and Content MathML (if MathML can represent all content dictionaries).
  • it must be posible to create presentational MathML, automatically derived from the semantics or influenced by the user. gemse must be able to synchronise semantics and presentation
  • keyboard control should be fully configurable
  • One should be able to create keyboard shortcuts and shortcut buttons while editing. This should be very dynamic. One should for example also be able to link the greek alphabet to the key G while editing.
  • Various XSLT stylesheets for conversions from OpenMath and MathML to other languages should be available. (Though this hasn't to do anything with gemse itself.)
  • It should be easy to change the layout of the elements of the user interface. (If gemse will use XUL, the user can simply edit the XUL description of the user interface.)
  • The content dictionary browser should be able to handle all content dictionaries. It probably has to fetch some of them directly from the internet.
  • The content dictionary browser also should be able to show the description of a content dictionary entry. This could be very useful as help for the user.
  • It would be very great if gemse could be part of Amaya.
  • Gemse should also be an authoring tool for OMDoc.

graphical interface

A short description of the parts that should be in the user interface:

  • Most important, on the top, a big view of the structure of the formula, i.e. the OpenMath document structure displayed as nested boxes.
  • On the bottom, the presentation of the formula, i.e. the rendering of the content MathML.
  • The bar on the right (and perhaps also one on the left) will be used to offer content dictionary entries, signs, operations on the semantic structure and operations on the presentational structure to the user.
